## 2024-05-14 - Improve accessibility for Kanban board empty states and generic action buttons
**Learning:** Generic, repetitive actions within lists or columns (like an "Add task" button at the top of each Kanban column) are challenging for screen reader users because they all sound identical ("Add task"). Additionally, generic `div` elements used as click targets (like an empty column state placeholder) are completely inaccessible to keyboard users and screen readers unless explicitly marked with `role="button"`, `tabIndex={0}`, an `onKeyDown` handler (Space/Enter), and focus ring styles.
**Action:** Always append structural context to the `aria-label` of repetitive buttons (e.g., `aria-label={"Add task to " + columnTitle}`). For custom interactive elements (like `div` placehoders), ensure they receive `role="button"`, `tabIndex={0}`, an `aria-label`, visible focus indicator styles (`focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2`), and an `onKeyDown` event listener that triggers the identical action when Space or Enter are pressed.
