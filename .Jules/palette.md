## 2024-03-21 - Accessible Add Task Buttons in Kanban
**Learning:** Interactive elements nested within distinct visual columns (like the "Add task" empty state div) need context-aware `aria-label`s (e.g., "Add task to To Do") to differentiate them for screen reader users, and must implement full keyboard accessibility (`role="button"`, `tabIndex={0}`, `onKeyDown` with `e.preventDefault()`, and visible focus rings) when implemented using non-native `<button>` elements like `div`.
**Action:** When implementing Kanban boards or similar multi-column layouts, ensure empty state actions are fully keyboard accessible and their accessible names dynamically include the column title context.
