## ğŸš¨ Common Mistakes to Avoid

### âŒ Don't:

1. **Send same generic cover letter everywhere**
   - Personalize with company name
   - Mention specific tech they use
   - Reference their product if possible

2. **Overwhelm with too much info**
   - Don't send all 5 documents
   - Choose: highlights for quick read, full study for detail

3. **Forget to test demo**
   - Always test in incognito before sending
   - Check it works on mobile
   - Verify no console errors

4. **Leave broken links**
   - Test every link before sending
   - Check GitHub repo is public
   - Verify demo is up (Render can sleep)

5. **Skip proofreading**
   - Run spell check
   - Have friend review
   - Read out loud

### âœ… Do:

1. **Tailor to each application**
   - Highlight relevant tech (if they use React, emphasize React patterns)
   - Match their terminology
   - Reference their values

2. **Make it scannable**
   - Use bullet points
   - Bold key achievements
   - Break into sections

3. **Quantify achievements**
   - "73% performance improvement"
   - "90% fewer API requests"
   - "< 5MB bundle size"

4. **Show, don't just tell**
   - Include GIFs
   - Link to live demo
   - Point to specific code

5. **Follow up**
   - If no response in 1 week, send gentle follow-up
   - Reference case study
   - Ask if they need any clarification
